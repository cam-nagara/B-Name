# B-Name 仕様定点観測 設計意図

> 本書は、B-Name における「仕様定点観測」の扱いそのものを定める補助文書である。
> 個別機能の設計意図や実装仕様そのものは、本書ではなく `docs/B-Name_設計意図.md` と `docs/B-Name-plan.md` を正本として扱う。

## 1. 前提

- グローバル共通ルール（`~/.codex/AGENTS.md`）には、`仕様定点観測` そのものの専用規定は存在しない
- そのため B-Name では、`D:/Develop/Meldex-public/app/docs/spec-observation-rules.md` を参照して運用ルールを継承する
- ただし対象プロダクトは B-Name であり、Meldex 固有の機能分類やフォルダ構成はそのまま流用しない

## 2. 文書の役割分担

- `docs/B-Name_設計意図.md`
  - ユーザーが望んでいる仕様意図の正本
  - ユーザー発言を優先し、粒度を丸めずに残す
- `docs/B-Name-plan.md`
  - 現行仕様、現行コード、未達差分、実装順を整理する計画書
  - ユーザー意図と現行コードの差がある場合は、その差分を明示する
- `docs/仕様定点観測_設計意図.md`
  - 仕様定点観測の運用方針自体を定める補助文書
- `docs/spec-snapshot-*`
  - 必要になった時だけ作る実態スナップショット履歴
- `docs/spec-snapshot-*-audit-*.md`
  - 設計意図と実態スナップショットの照合レポート履歴

## 3. 既定の運用

### 3.1 実行条件

- 仕様定点観測の各作業は、**ユーザーの明示的な指示があった時だけ** 実行する
- 指示が無い限り、定期的な自動更新や一括再生成はしない

### 3.2 分離する3作業

- **設計意図更新**
  - ユーザーが何を望んでいるかを更新する
- **実態スナップショット**
  - その時点のコード実態を記録する
- **照合**
  - 設計意図と実態スナップショットの差を整理する

この 3 作業は混同せず、必要時のみ個別に行う。

### 3.3 差分更新の原則

- 既存文書を必ず先に読む
- `反映済みログ範囲` より後の差分を一次探索対象にする
- 差分で整合が保てる場合は差分更新に留める
- `反映済みログ範囲` が曖昧、または既存文書に取りこぼしが多い場合は全量見直しに切り替える

## 4. 情報源の優先順位

仕様判断の優先順位は以下とする。

1. 同一論点についての**最新のユーザー発言**
2. それ以前のユーザー発言
3. `docs/B-Name_設計意図.md`
4. 現行コード
5. `docs/B-Name-plan.md`
6. 補助調査資料

補足:

- ユーザー発言同士が矛盾する場合は、**最終判断を優先**する
- ユーザーが未指定の細部は、現行コードを参考に計画書側で補う
- 現行コードがユーザー意図に反している場合、設計意図側ではユーザー意図を正とし、計画書側では「未達差分」として明記する

## 5. B-Name で特に重視する対象

現時点の B-Name では、次を仕様定点観測の重点対象とする。

- 用紙設定
- ページ一覧と見開き
- コマ枠作成・枠線編集
- コマ編集モードと紙面編集モードの往復
- コマプレビュー反映
- PSD 書き出し
- `.clip` 直書き検討の延期判断

## 6. 更新単位

- 原則は **B-Name 全体ではなく、現在会話で差分が出た機能単位** で更新する
- ただし、今回のように複数機能へまたがる仕様変更が連続している場合は、関連範囲をまとめて更新してよい
- 全機能の全量見直しは、ユーザーが明示した時だけ行う

## 7. 履歴の扱い

- 設計意図文書と計画書は上書き更新してよい
- ただし `spec-snapshot-*` と照合レポートは履歴資料として扱い、**後から上書きして過去の判断を消さない**
- 同日再作成が必要な場合は `-rev2`, `-rev3` を付ける

## 8. 今回時点の適用判断

- B-Name では、仕様定点観測の「意図の正本」は `docs/B-Name_設計意図.md` とする
- `docs/B-Name-plan.md` は、意図仕様そのものではなく、**現行コードと実装計画を含む整理文書** として扱う
- コマ編集モード、用紙色、開始ページ、枠線ツール、PSD 書き出しは、直近の変更量が大きく、今後も差分監視対象とする

## 9. 今後の更新トリガ

以下のような指示が来た時に、本運用を使う。

- `設計意図を更新して`
- `実態スナップショットを更新して`
- `照合して`
- `仕様定点観測をして`

これらの指示が無い通常の実装セッションでは、本書は参照するだけに留める。
