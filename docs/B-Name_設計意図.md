## 0.6 2026-04-26 時点の追加設計意図（競合時はこの節を優先）

### 0.6.1 仕様定点観測の扱い
- B-Name でも `仕様定点観測` を採用する
- ただしグローバル共通ルールには専用規定が無いため、`Meldex-public` 側の運用を参照し、B-Name では `docs/仕様定点観測_設計意図.md` を基準に扱う
- 今後は **設計意図**（ユーザーが望む仕様）と **計画書**（現在の仕様整理・未達差分）と **実装実態スナップショット** を分離して残す

### 0.6.2 コマ編集モードの意図仕様
- **1コマ = 1 `panel_NNN.blend`** の独立ファイル方式は維持する
- ただし、コマ編集モードに入っても **作品共通の用紙設定は崩れてはいけない**
  - 開始ページの位置
  - 読む方向
  - 用紙色
- コマ編集モードでは **そのコマだけが完全に孤立して見える状態は不可**
- 少なくとも以下は参照表示できる必要がある
  - 同じページの他コマ
  - 他のページ
  - 紙面上の位置関係
- 参照表示は読取専用でよいが、**構図合わせの基準として常時見える** ことを優先する

### 0.6.3 コマ編集結果の紙面反映
- コマ編集モードを閉じてページ一覧・紙面編集へ戻った時、**そのコマの編集内容は即座に紙面側へ見える** 必要がある
- 反映手段はサムネイルでもプレビューでもよいが、「保存したのに紙面では古いまま」は不可
- 自動更新の既定は軽量な方式でよいが、ページ表示・書き出し・後続編集の3箇所で同じ最新内容を参照できる必要がある

### 0.6.4 枠線ツールの最新意図
- 枠線カットツール / 枠線選択ツールのモーダル中でも、B-Name パネルやビューポート右上ナビゲーションは操作できる
- 枠線選択ツールでは以下を直接編集できる
  - 枠線全体
  - 1辺だけ
  - 1頂点だけ（接続2辺へ適用）
- `コマ:枠線` セクションの対象は、アクティブコマではなく **枠線選択ツールで選択した枠線** を優先する
- 枠線の `▲` 拡張は、隣接候補へぴったり重なる方向の操作で **コマ形状を壊してはならない**
- 縦横均等分割は枠線ツール文脈で使う。対象は選択中コマで、矩形だけでなく変形コマにも対応する

### 0.6.5 用紙設定の最新意図
- 用紙パネルから **用紙色を直接編集可能** にする
- 開始ページの初期値は **左から**
- セーフライン外側の塗りつぶし色の新規作品初期値は **明度 0.7**

### 0.6.6 書き出しの最新意図
- 実用優先の主目標は **PSD 書き出し強化**
- PSD では、コマは **コマ形状のレイヤーマスクが掛かったレイヤーフォルダ** として出ることを優先する
- Grease Pencil / フキダシ / 効果線 / テキスト / 画像レイヤーも、可能な限り個別レイヤーとして残す
- `.clip` 直書きは現時点では見送る
  - コマフォルダ化やラスターレイヤー化の可能性は調査済み
  - ただし工数と破損リスクを考え、今は計画書だけ残して着手しない

## 0. プロジェクト全体の位置付け

### 0.1 B-Name とは
- Blender上で動作する、漫画ネーム作画専用のアドオン
- CLIP STUDIO PAINTの3D機能と同等以上の体験を、Blenderの3Dエンジン上で実現する
- 3D操作と2D作画（コマ/フキダシ/ペイント）をシームレスに統合する

### 0.2 Meldexとの関係（独立性）
- **独立したアドオンとして開発する**
- 連携はMeldex→B-Name一方向のみ
  - Meldex → B-Name: 「クリスタへ送信」と同様の「B-Nameへ送信」機能でシナリオ（台本テキスト＋ページ構成）を送信
  - B-Name → Meldex: 連携なし
- Meldexのソースコードには一切依存しない
- Meldex側には「B-Nameへ送信」ボタンの追加のみ行う（別計画書で管理）

### 0.3 なぜBlenderアドオンなのか
- Blenderビューポートをブラウザ（Meldex）に埋め込む公式手段は存在しない
- 3D操作と2D作画のシームレス統合にはBlender描画スタックへの直接アクセスが必須
- Grease Pencilオブジェクトは3D空間内で2D描画を行う既存機能で、ネーム作画の基盤として最適

### 0.4 Pythonの速度・UI懸念への方針
- 描画処理本体はBlenderのC++エンジンが担うため致命傷にならない
- UIは Blender標準UI（Panel / Operator / Popup / Modal）で必要十分、不足時は `gpu` モジュールでオーバーレイ描画
- 重いPython処理はC拡張化を後段で検討（MVPでは不要）

## 1. 階層構造

### 1.1 基本階層
- 作品 ＞ ページ ＞ コマ の**3層で確定**
- 「プロジェクト」という呼称は廃止、「作品」に統一する
- 長期連載作品共通の設定（作品より上の階層）は**一旦不要**

### 1.2 実体
- 作品: `MyWork.bname/` ディレクトリ（拡張子風フォルダ名）
- ページ: `pages/NNNN/` 配下の1ディレクトリ
- コマ: `panel_NNN.blend`（独立ファイル）

### 1.3 メタデータファイル
- `work.json`: 作品全体メタデータ
- `pages.json`: ページ一覧・順序管理（ページ並び替えはこのJSON順序変更のみで完結）
- `page.json`: 個別ページメタデータ
- `panel_NNN.json`: 個別コマメタデータ
- スクショサムネイル画像（`panel_NNN_thumb.png`）／手動高品質プレビュー画像（`panel_NNN_preview.png`）はバックアップ対象外でよい

## 2. ファイル方式

### 2.1 独立コマファイル
- **1コマ=1.blendファイル**として独立させる
- 理由: メモリ効率／コマの移動・複製が容易／バージョン管理との相性／破損の局所化
- コマファイルは3Dモデル本体を持たず、アセットライブラリからのリンク参照のみで構成

### 2.2 ファイル命名規則
- ページディレクトリ: `pages/NNNN/`（4桁ゼロパディング）
- コマファイル: `panel_NNN.blend` / `panel_NNN.json` / `panel_NNN_thumb.png` / `panel_NNN_preview.png`（3桁ゼロパディング）
- 内部ファイル名は常に連番固定（リネームしない）
- ユーザー向け表示名は `panel_NNN.json` に別途保持（表示名変更で参照が壊れないようにする）

### 2.3 コマの移動・複製
- コマを他のページへ移動・複製できる
- ファイル単位で完結する（ファイル移動/コピー）
- 移動先でファイル名衝突したら自動リネーム（空き番号へ）
- 作画レイヤー（Grease Pencil・フキダシ）もコマと同時に移動・複製される

## 2.5 基本作画機能

### 2.5.1 レイヤー機能（2種類のレイヤーを扱う）
- **ベクターレイヤー**（Grease Pencil v3ベース）— 線画・フキダシ・塗り
- **画像レイヤー**（ビットマップ）— スキャンラフ取り込み・写真参照・実写背景

共通機能:
- レイヤー追加・削除・複製・並び替え・結合
- 表示／非表示・ロック・不透明度・ブレンドモード
- レイヤーフォルダ（Grease Pencil v3 の `GreasePencilLayerGroup` を活用）

### 2.5.2 画像レイヤー固有機能
- 画像ファイル読み込み（PNG / JPG / TIFF / PSD）
- 位置・拡大縮小・回転・反転
- 不透明度・ブレンドモード
- 簡易レベル補正（明度／コントラスト／2値化）— 下書き取り込み用途
- 描画方式は `draw_handler_add` + `gpu` オーバーレイで統一（平面メッシュは使わない、Grease Pencil v3 画像リファレンスも不採用）

### 2.5.3 ペイント機能
- ペン／鉛筆／マーカー／消しゴム／塗りつぶし
- ブラシサイズ・不透明度・**筆圧**
- カラーピッカー（Blender標準 ColorRamp を活用）
- 定規・直線・図形ツール

### 2.5.4 ブラシ方針（段階実装）
- **Grease Pencil v3 標準ブラシ**で漫画的線質（入り抜き・筆圧・テーパー）をまず構築
- 不足する質感（かすれ・アナログ感）は Phase 3 で**独自ブラシプリセット**として追加
- 独自ブラシエンジンは Phase 6 以降に後送り

### 2.5.5 コマ割り機能（基本）
- コマ枠の自動生成: **縦N分割／横N分割／指定レイアウト**
- コマ枠のドラッグ編集
- コマごとにローカル座標系を持ち、その中で3Dモデル配置・作画が可能
- **コマ境界でのクリッピング**（コマ領域外の線は自動で隠す）

## 3. 3D連携

### 3.1 2層構造（紙面ワールド / 3Dワールド）
- 紙面シーン: 原稿用紙・コマ枠・Grease Pencilレイヤー・画像レイヤー・フキダシを保持
- コマごとの3Dシーン: 各コマが独自の3D空間を持つ
- **同一ページ内の異なるコマで、まったく別の場所・別の登場人物を扱える**（長期連載の話数間も共有/非共有で自由）

### 3.2 カメラ構成
- **カメラリグ方式**: 親Empty + 子カメラ
- パン・ズーム → 親Empty
- 画面回転（ロール） → 子カメラ（方法A確定）
- B-Nameの紙面編集モード中およびコマ編集モード中は、ビューポートを常にカメラビューに固定
- 画面回転ではコンテンツは動かず、カメラが傾くことで結果的に画面上で全要素が一緒に回って見える

### 3.3 モード切り替え
- **紙面編集モード**: 紙面シーンで作画。コマ内の3Dはプレビュー表示
- **コマ編集モード**: コマのダブルクリックまたは右クリックで入る。アクティブシーンがそのコマの3Dシーンに切り替わる
- コマ編集モードへ入っても、**用紙色／開始ページ／読む方向など作品共通の用紙・ページ配置設定は紙面編集モードと一致したまま**でなければならない
- コマ編集モードで編集対象として直接操作できるのはそのコマ自身だけに限定するが、**参照表示として同一ページの他コマ・前後ページ・見開き相手ページを読み取り専用で見られる**ようにする
- 「そのコマ以外が一切見えない孤立した空シーン」は採用しない。構図合わせ・連続性確認・視線誘導の確認ができることを優先する

### 3.4 コマ枠内の3D表示（スクショサムネイル方式）
- **「コマを閉じるたびにレンダリング」は避けたい**という方針
- レンダリングは書き出し時のみ
- コマ編集モード終了時: **ビューポートのスクリーンショット**を `panel_NNN_thumb.png` に保存（数十ms、レンダリングなし）
- 紙面編集モードへ戻った時は、**直前に編集したコマの内容がページ一覧・紙面上に即反映される**こと。直近スクショがそのままページ側の可視状態の基準になる
- 過去スクショがあればそれを表示
- **新規コマでスクショ未生成時: コマ枠の背景色（白）のみ**（プレースホルダ画像は生成しない）
  - 理由: 新規コマ作成直後はどのBlenderファイルを読み込むかも決まっていないため、スクショ取得は原理的に不可能
- スクショ解像度はコマ枠の表示サイズに合わせた最小限
- ユーザー手動「高品質プレビュー生成」: コマ右クリックから、そのコマだけ低解像度レンダリングし `panel_NNN_preview.png` に保存
- `panel_NNN_preview.png` と `panel_NNN_thumb.png` の両方がある場合は、更新日時が新しい方を使う。古い高品質プレビューが編集後の自動サムネイルを隠してはいけない

### 3.5 3Dアセット管理
- **Blender標準のアセットブラウザを活用する**（B-Name独自のアセット管理UIは作らない）
- アセットライブラリは**2層構造**:
  1. グローバルアセットライブラリ（全作品共通）
  2. 作品固有アセット（`MyWork.bname/assets/`）
- 作品単位の中間層は作らない（長期連載共通設定と同じく不要）
- 3D素材は作品に関係なく全作品で共有したい

### 3.6 モデル本体の編集
- **3Dモデル本体の編集はBlender本体で直接 `.blend` を開いて行う前提**
- B-Name上ではリンク参照したモデルのアセンブル（配置・ポーズ）のみ
- カスタム右クリックメニュー「リンク元ファイルを開く」で、**新しいBlenderインスタンスを subprocess で起動**（現在の編集作業を失わない）
- 設定で「現在のBlenderで開く」にも切替可能

### 3.7 3D編集機能
- **3Dモデルのポーズ編集**はコマ単位で独立（他コマに影響しない）
- **3Dモデルの線画化**（Freestyle / Grease Pencil変換）
- **3D→2D投影して作画レイヤーへ落とし込み**

### 3.8 非採用機能
- **パース定規機能は実装対象外**（3Dカメラがパース情報を直接持つため不要）
- **トーンは不要**

## 3.9 Meldex連携（受信）

- Meldex側の「B-Nameへ送信」ボタンでシナリオをPOST送信

## 4. 用紙設定

### 4.1 既定値
- **解像度初期値: 600dpi**
- **単位**: mm（既定）／ピクセル／inch の切替可
- 用紙サイズプリセット: B4 / A4 / B5 / 同人誌判 / 商業判 / カスタム
- **開始ページの既定値は「左から」**
- 原稿用紙の他の要素の初期値はCLIP STUDIO PAINTの「集英社マンガ誌汎用（少年誌・少女誌・青年誌共通/新書・B6判単行本化）」と同じ
  - キャンバス 257×364mm
  - 製本（仕上がり） 221.81×328.78mm
  - 裁ち落とし 7.00mm
  - 基本枠（内枠） 180×270mm
  - セーフライン 天17.49 / 地17.49 / ノド20.90 / 小口17.23mm
  - 基本表現色 モノクロ、基本線数 60.00、用紙色 白
  - カラープロファイル（表示用） sRGB IEC61966-2.1
  - ノンブル フチ幅 0.30mm、ノンブル 基本枠との間隔 縦5.00mm / 横0.00mm

### 4.2 プリセット機能
- **CLIP STUDIO PAINTと同様にプリセットを登録・切り替えできる**
- 既定プリセット「集英社マンガ誌汎用」を同梱
- プリセット保存場所: グローバル／作品ローカルの2層
- 用紙色は**プリセット経由だけでなく、用紙パネルから直接編集可能**にする

### 4.3 セーフライン外側オーバーレイ
- **セーフラインの外側を、指定色で塗りつぶし、乗算で全体に重ねる**
- ON/OFF切り替え
- 既定色は**明度 0.7 のグレー**（`#B3B3B3` 相当）、不透明度 30%、ブレンドモード 乗算
- 表示専用（書き出しには含めない）
- ページ単位でオーバーライド可能

## 5. ページ管理

### 5.1 基本機能
- ページ一覧パネル（サムネイル表示）
- ページ追加・削除・並び替え・複製
- ページ間ジャンプ

### 5.2 見開き変更
- **ページを見開きに変更できる**（CLIP STUDIO PAINTの「見開きに変更」相当）
- ダイアログ:
  - 「NページとMページを見開きとして結合します」の情報表示
  - 「対象ページを開いている場合は編集中の内容でファイルに保存されます。この動作は[取り消し]を実行できません」の警告
  - **トンボを合わせる**（ON/OFF、既定ON、間隔既定 `-9.60mm`）
  - **描画されていないレイヤーを削除**（ON/OFF、既定ON）
- 見開き解除も可能
- `pages.json` で `spread: true` エントリとして保持、`originalPages` に元ページIDを記録
- 見開きディレクトリは `pages/NNNN-MMMM/` のように結合元番号を連結

## 6. コマ枠機能

### 6.1 形状
- 長方形／多角形／曲線（ベジェ）／フリーフォーム
- **コマ枠同士の重なりを許容**（飛び出しコマ／挿入コマ／アイキャッチ等）

### 6.2 枠線スタイル
- 線種: 実線／破線／点線／二重線／カスタム
- 線幅（mm）、線色
- 角処理: 直角／丸角／面取り
- **辺ごとの個別オーバーライド**（上辺だけ太線、右辺だけ破線、等）
- **枠線全体の非表示**（コマ機能は維持したまま）

### 6.3 白フチ
- ON/OFF、幅（mm）、**既定 0.37mm**
- 色（既定: 白）
- 枠線の**外側**に描画
- 辺ごとの個別オーバーライド

### 6.4 作成・編集ツール
- 矩形コマ作成
- **縦横均等分割**は独立した「分割テンプレート」セクションではなく、**枠線ツール内**に置く
- 縦横均等分割の対象は「現在アクティブなページ全体」ではなく、**枠線選択ツールで選択中のコマ**とする
- 縦横均等分割は**矩形コマだけでなく変形コマ（polygon）にも対応**する
- **枠線カットツール**（CLIP STUDIO PAINTの同名ツール相当、ドラッグで切断、切断後に作品共通のコマ間隔が自動適用）
- 水平カット／垂直カットの個別ボタンは不要
- **枠線選択ツール**で、辺単位／頂点単位／枠線全体単位の選択と編集を行える
- 枠線選択ツールで選択した辺の両端頂点は、選択ハイライトとして表示する
- 枠線選択ツール・枠線カットツールの使用中でも、**B-Name パネル操作とビューポート右上ナビゲーション（虫眼鏡・手のひら等）は使える**ようにする
- 枠線選択ツール内のハンドルクリック1回、ドラッグ確定1回を**それぞれ独立してアンドゥ／リドゥ**できるようにする
- 枠線の色・太さは、コマ全体だけでなく**1 辺だけ／1 頂点に接続する 2 辺だけ**にも設定できる
- 頂点編集、結合・再分割、フリーフォーム

### 6.5 スナップ
- **仕上がり枠・基本枠・セーフライン・トンボにスナップする**
- **他のコマ枠にもスナップする**（作品共通のコマ間隔を保って）
- Shiftで一時無効化

### 6.6 コマ間隔ルール（作品内共通）
- **上下の間隔・左右の間隔を別々に設定可能**
- mm単位
- **既定: 上下 7.3mm / 左右 2.1mm**
- **作品内共通ルールとして共有**（`work.json` に保存、作品内全ページ・全コマで共有）
- コマ単位で個別オーバーライド可

### 6.7 Z順序・重なり
- 各コマはZ順序値を持つ
- **重なり順を変えられる**（最前面へ／最背面へ／1つ前面へ／1つ背面へ、コマ一覧ドラッグ）
- **自動くり抜き 既定ON**（上位Zのコマが重なる範囲は下位Zのコマから自動くり抜き）
- コマ単位でくり抜かない設定にも切替可能
- ぶち抜きコマ（仕上がり枠超え・裁ち切りまで）も同じ仕組み

## 7. フキダシ・テキスト

### 7.1 フキダシ形状（本体）
**Meldexボードのカード形状と完全一致で実装する**:
- 矩形
- 楕円
- 雲
- トゲ（曲線）
- トゲ（直線）

**5形状で確定。**（角丸・付箋型しっぽは形状プリセットではない）

### 7.2 角丸オプション
- フキダシ形状ではなく**全形状共通のオプション**
- ON/OFF、角半径（mm）
- 楕円のように既に丸い形状では無効

### 7.3 カスタムフキダシ形状
- **パスツールで作成した任意の形状をフキダシとして登録できる**
- 名前を付けて保存、縮尺方式（絶対／相対）を登録時選択
- 登録後の頂点再編集・削除・リネーム可
- 保存場所: 作品ローカル `MyWork.bname/assets/balloons/` ／グローバル

### 7.4 尻尾（しっぽ）
- **全形状共通で追加できるオプション**
- 3種類:
  - 直線尻尾（三角形）
  - 曲線尻尾（ベジェで膨らませる）
  - 付箋型しっぽ（Meldex注釈ツールの付箋のような矩形タブ）
- 各尻尾のパラメータ: 方向／長さ／根元幅／先端幅、曲線尻尾ではカーブ曲がり具合
- 1フキダシに複数尻尾対応（話者共有セリフ・ハモり等）
- フキダシなしテキストにも追加可（擬音から線を引き出す等）

### 7.5 フキダシなしテキスト
- **フキダシ本体を持たないテキスト単体オブジェクトも作成可能**
- 用途: 擬音／手書き風の台詞／ナレーション／心の声／場面転換の文字等
- テキスト属性はフキダシ内テキストと共通

### 7.6 テキスト属性
- フォント、フォントサイズ、行間、字間、縦/横書き切替
- ルビ、縦中横
- 文字色
- **白フチ（テキストストローク）**: 指定幅（mm）でテキスト外周を縁取り
  - 幅既定 0.2mm、色既定 白
  - ON/OFF
  - フキダシ内テキスト・フキダシなしテキスト両方で利用可能

### 7.7 縦書き日本語組版
- **縦書きを実現する**
- **ルビ**（親文字＋ルビ文字、モノルビ／グループルビ／熟語ルビ）
- **縦中横**（半角数字・英字の縦書き内横向き表示）
- **行間（行送り）**設定可能
- **字間（字送り）**設定可能
- 禁則処理（行頭禁則／行末禁則／ぶら下げ）
- 字形選択（縦書き用グリフへの自動切替）

### 7.8 セリフ種別→形状マッピング
Meldexから受信したセリフ種別を既定形状にマッピング（作品単位で変更可）:
- 通常セリフ → 矩形（角丸ON）
- 思考 → 雲
- 叫び → トゲ（直線）
- ナレーション → 矩形（角丸OFF）

## 9. ビューポート操作ショートカット

**CLIP STUDIO PAINTのショートカットを実装する:**

| 操作 | ショートカット |
|---|---|
| 手のひら（パン） | `Space + ドラッグ` |
| 回転 | `Shift + Space + ドラッグ` / `Shift + マウスホイール` |
| ズームイン | `Ctrl + Space + クリック`（macOS: 先にSpace→Cmd） |
| ズームアウト | `Alt + Space + クリック` |
| ズーム（連続） | `Ctrl + Space + ドラッグ` / `マウスホイール` |
| レイヤー選択 | `Ctrl + Shift + クリック` または `Ctrl + Shift + ドラッグ` |
| スポイト | `右クリック` |

- B-Name専用キーマップとして独立定義、アドオン設定でカスタマイズ可
- Blender標準操作（中ボタンドラッグ等）も従来通り併用可能
- B-Nameモード中のみ既定キーマップを一時オーバーライド（KeyMapItem.active切替）

### 9.1 モード依存の挙動
- **紙面編集モード中**: ショートカットは紙面シーンのビュー用カメラに作用
- **コマ編集モード中**: ショートカットはそのコマの3Dシーンのシーンカメラに作用
- **ペイント中**: すべてのショートカットが優先（ストローク描画より先に判定）
- **オブジェクト選択中**: レイヤー選択は作画レイヤーのみを対象、3Dオブジェクト選択はBlender標準キー（左クリック）を維持
- **テキスト編集中**（フキダシ内文字入力）: ビューポート操作ショートカットは一時無効化

## 10. 作品情報・ノンブル・書き出し

### 10.1 作品情報（原稿上表示）
- **作品情報を原稿上に表示したい**
- 作品名／話数／サブタイトル／作者名
- 表示位置: 上／下 × 左／中央／右 の9通り（項目ごとに個別指定）
- フォント、サイズ、色
- 見開きページでの左右個別配置
- 書き出し時にも原稿画像に焼き込まれる

### 10.2 ノンブル（ページ番号）
- ON/OFF、開始番号（既定1）、フォーマット（プレースホルダ `{page}`）
- フォント既定 `I-OTFアンチックStd B`、サイズ既定 9.0pt
- 位置、基本枠との間隔（縦5.00mm / 横0.00mm 既定）
- フチをつける（ON/OFF、フチ幅既定 0.30mm）
- 隠しノンブル（ON/OFF、裁ち落とし外へ配置）

### 10.3 単一ページ書き出し
- 現在開いているページ（または選択した1ページ）の書き出し
- フォーマット: PNG / JPEG / TIFF / PSD / PDF
- カラーモード: モノクロ／グレースケール／RGB／CMYK
- 出力範囲: 仕上がり枠／裁ち落とし込み／基本枠／キャンバス全体
- 解像度・倍率指定
- **PSD はレイヤー保持で出力**し、各コマはフォルダ化して扱う

### 10.4 複数ページ一括書き出し
**CLIP STUDIO PAINTの「複数ページ書き出し＞一括書き出し」のように、作品内の全ページもしくは選択した複数ページを連番統合縮小画像出力できるようにする**:
- 対象: 全ページ／選択ページ／範囲指定
- 見開きページは1枚の画像として書き出し（または左右分割も選択可能）
- 出力形式: PNG/JPEG/TIFF/PSD/PDF
- カラーモード: モノクロ／グレースケール／RGB／CMYK
- 連番ファイル名テンプレート（プレースホルダ `{workName}` / `{episode}` / `{page}` / `{pageStart}-{pageEnd}`）、ゼロパディング桁数指定
- 縮小出力: 倍率／最長辺ピクセル／解像度指定
- **補間方法**: バイキュービック／ランチョス／バイリニア／ニアレストネイバー
- 出力範囲: 仕上がり枠／裁ち落とし込み／基本枠／キャンバス全体
- 追加オプション: コマ枠線／白フチ／トンボ／ノンブル／作品情報／用紙色の含める/含めない、レンダーキャッシュ使用／強制再レンダリング
- セーフライン外側オーバーレイは**常に含めない**
- バックグラウンド処理、進捗ダイアログ（「Nページ中Mページ処理中」）、キャンセル可
- エラー発生ページはスキップして続行、最後にエラー一覧表示

#### 10.4.1 PSD 出力のレイヤー方針
- コマごとに**コマフォルダ**を作る
- コマ形状のレイヤーマスクは、コマフォルダ内の**content サブフォルダ**に掛ける
- 白フチと枠線は、マスクで切れないよう content とは分けて保持する
- Grease Pencil、効果線、フキダシ、テキストは**ラスターレイヤー**として出力する
- `.clip` 直接書き出しは当面の主目標にせず、**PSD を優先**する

### 10.5 書き出し出力先・履歴
- **既定出力先**: `MyWork.bname/exports/<出力日時>/`
- 書き出し履歴は `work.json` の `lastExport` に記録（タイムスタンプ／使用プリセット／ページ数／出力ディレクトリ）

### 10.6 書き出しプリセット
- よく使う設定を名前付きで保存・呼び出し
- `work.json` の `exportPresets` に格納
- **既定プリセット（同梱）3種**:
  - 「印刷入稿用PNG」（600dpi / 裁ち落とし込み / モノクロ）
  - 「確認用縮小PNG」（長辺1600px / RGB）
  - 「PDF結合（全ページ）」

### 10.7 書き出しパイプライン（オーバーレイ要素の焼き込み）
`draw_handler_add` によるビューポートオーバーレイは `bpy.ops.render.render()` の出力画像には焼き込まれないため、書き出し時は Pillow で再描画して合成する:

1. 各コマの3Dレンダリング（EEVEE-Next / Workbench）
2. 紙面ベース画像の生成（Pillowで用紙サイズの白画像）
3. 画像レイヤー合成（Pillow）
4. コマ内レンダー画像の貼り込み（Z順序・重なりくり抜きも反映）
5. Grease Pencil v3 レイヤーの合成
6. コマ枠線・白フチの描画（Pillowベクター描画）
7. フキダシ・テキストの描画（Pillow + fontTools）
8. 作品情報・ノンブルの描画
9. トンボの描画（オプション）
10. セーフライン外側オーバーレイは含めない
11. カラーモード変換（モノクロ／グレースケール／CMYK は littleCMS でICCプロファイル変換）
12. ファイル保存（PNG/JPEG/TIFF/PSD/PDF）

書き出し時の描画関数とビューポート用 `draw_handler` 関数は同じ計算ロジックを共有するよう設計し、共通層を用意する。

## 11. 技術実装方針（Blender 4.3+ / 5.x基準）

### 11.1 対応バージョン
- 下限: Blender 4.3.0（Grease Pencil v3正式導入版）
- 主ターゲット: 5.x（最新メジャー）

### 11.2 配布形式
- **Extensions Platform対応**（`blender_manifest.toml` を正、`bl_info` は最小）
- `wheels/` フォルダに同梱ライブラリを配置

### 11.3 描画方式
- コマ枠画像・画像レイヤー・オーバーレイ要素（セーフライン外側／コマ枠線／白フチ／作品情報／ノンブル）は `SpaceView3D.draw_handler_add` + `gpu` モジュールで統一
- 書き出し時は Pillow で同じロジックを再描画して画像合成（`draw_handler` の出力はレンダリング画像に焼き込まれないため）

### 11.4 組版エンジン
- 3層構成:
  - ビューポート表示: `blf`
  - 書き出し再描画: `Pillow`
  - グリフ選択・メトリクス: `fontTools`
- Blenderテキストオブジェクトは使わない（縦書き・ルビ非対応のため）

### 11.5 スクショ取得
- 第1候補: `bpy.ops.screen.screenshot_area()` + `temp_override`
- フォールバック: `gpu.types.GPUOffScreen.draw_view3d()`

### 11.6 Meldex受信
- ローカルHTTPサーバー（localhost限定、既定ポート 47817）
- 別スレッドで `http.server.ThreadingHTTPServer` を起動、受信JSONは `queue.Queue` に積む
- `bpy.app.timers.register` でメインスレッドに戻して `bpy.data` を更新
- アドオン無効化時は確実にshutdown + join + timer解除

### 11.7 Grease Pencil
- v3 前提（`bpy.data.grease_pencils_v3` / `GreasePencilLayer` / `GreasePencilLayerGroup` / `GreasePencilDrawing`）
- レイヤーフォルダは v3ネイティブの `GreasePencilLayerGroup` を使用（独自実装しない）

### 11.8 subprocess起動
- `bpy.app.binary_path` で取得した実行ファイルを `subprocess.Popen` で起動
- Windows: `CREATE_NO_WINDOW` でコンソール窓抑制
- macOS: `.app` バンドル内実行ファイルを直接起動

### 11.9 画面回転
- B-Nameモード中は常にカメラビュー（`region_3d.view_perspective = 'CAMERA'`）に固定

### 11.10 キーマップオーバーライド
- B-Nameモード中のみ既定 `KeyMapItem.active = False` に切替、モード終了時に復元
- Blender Preset差異を検出して対応マッピングを選択

### 11.11 アセットブラウザ連携
- `bpy.types.FileHandler.poll_drop`（D&D公式フック）
- フォールバック: `bpy.app.handlers.depsgraph_update_post` で新規Object出現を検知

### 11.12 モジュール分割
- 計画書 5.1 の分割に従う
- register順序: preferences → core → io → typography → operators → panels → ui → keymap（unregister逆順）
- 1ファイル1000行制限、1関数50行目安（グローバルCLAUDE.md準拠）

### 11.13 同梱ライブラリ
- `fontTools`（MIT）、`Pillow`（HPND）
- PDF結合: `pypdf`（Phase 6b）
- PSD書き出し: `psd-tools` / `pytoshop`（Phase 6c）
- CMYK: `littleCMS` + ICCプロファイル（Phase 6d）

## 13. 判断の履歴（撤回された案は記録しない、最終判断のみ）

以下は議論途中で一度検討されたが、最終的に**採用された**方針:

- シーン方式ではなく**独立ファイル方式**（1コマ=1.blend）を採用
- アセットライブラリは**2層**（グローバル／作品固有）。3層案（グローバル／作品単位／プロジェクト固有）は廃止
- コマ閉じるたびのレンダリングは廃止し、**スクショサムネイル方式**に変更
- 新規コマのプレースホルダは**背景色白のみ**（画像生成なし）
